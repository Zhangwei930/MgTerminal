# 변경 로그


## [0.5.25] - 2026-07-22

### 기능
- **데스크톱 펫**: 설정 → AI → 펫에서 켜면 화면 어디든 드래그 가능한 떠다니는 펫이 나타나 AI 상태에 맞춰 애니메이션을 재생합니다 — 대기 중엔 숨쉬기, 실행 중엔 통통 튀기, 승인 대기 중엔 흔들기, 완료 시 손 흔들기, 실패 시 떨기. 펫은 시스템 포커스를 거의 갖지 않으므로 포커스 없는 창에 대한 Electron의 애니메이션 제한을 이 창에서만 명시적으로 해제해 멈춘 것처럼 보이지 않게 했습니다
- **클릭으로 바로 이동**: 클릭하면 AI 채팅 패널을 열거나 포커스하며, 가능하면 현재 작업 중인 터미널 세션 탭으로 이동합니다. 더블클릭은 메인 창을 포커스합니다. 우클릭 메뉴에서 설정에 등록한 명령 실행, AI 설정 열기, 펫 위치 초기화, 펫 숨기기를 할 수 있습니다. 마우스를 올리면 더 자세한 상태 말풍선이 표시됩니다
- **외형 커스터마이징**: 직접 이미지나 스프라이트 시트를 업로드할 수 있으며, 스프라이트 시트는 상태별(대기/실행/대기승인/완료/실패)로 재생 프레임 범위를 지정할 수 있습니다. 크기, 불투명도, 항상 위, 말풍선 표시 여부를 조절할 수 있습니다
- **프라이버시 모드와 완료 알림**: 프라이버시 모드에서는 말풍선에 실행 중인 도구 이름 대신 "실행 중"과 같은 일반적인 상태만 표시합니다. 10초 이상 걸린 작업이 완료되거나 실패하면 선택적으로 데스크톱 알림을 띄울 수 있습니다
- **위치 저장과 멀티 모니터 지원**: 드래그한 위치는 재시작하거나 펫을 다시 켜도 유지됩니다. 모니터 연결이 끊기거나 해상도가 바뀌어 펫이 화면 밖으로 나가면 자동으로 기본 모서리로 되돌아갑니다

## [0.5.24] - 2026-07-22

### 수정
- **상태 점검 프로브가 지금까지 키 파일을 한 번도 실제로 읽은 적이 없었습니다**: 개인 키를 읽는 비동기 함수를 호출할 때 await가 빠져 있어서, 실제 파일 내용이 아니라 아직 처리되지 않은 Promise를 검사했고 그 결과 매번 "개인 키가 아님"으로 판정되어 조용히 건너뛰었습니다. 인라인으로 저장된 키가 아니라 로컬 키 파일에 의존하는 호스트는 터미널 연결은 멀쩡해도 상태 점검에서는 반드시 인증에 실패했습니다
- **이 기기에서 복호화할 수 없는 자격 증명을 더 이상 로그인 거부로 보고하지 않습니다**: 여전히 암호화된 자리표시자인 비밀번호나 키는 프로브 전에 빈 값으로 제거되어, 서버는 자격 증명이 전혀 없는 로그인을 당연히 거부했습니다. 이제는 "자격 증명은 설정되어 있지만 이 기기에서 복호화할 수 없음"을 판별해 Vault 잠금 해제나 보안 저장소 복구를 안내합니다
- **호스트 키가 아직 신뢰되지 않은 경우를 인증 실패로 잘못 보고하지 않습니다**: 프로브는 호스트 키가 알 수 없거나 변경되었을 때 이미 모든 인증 방법을 보류하고 있었지만, 그 사실을 패널에 전달한 적이 없었습니다. 이제는 전용 "호스트 키 미검증" 상태를 표시하고, 신뢰를 확립하려면 한 번 수동으로 연결하라고 안내합니다
- **"암호화된 키를 건너뜀" 알림이 더 이상 운에 좌우되지 않습니다**: 이전에는 어떤 인증 방법도 시도되지 않았을 때만 표시되었지만, SSH 에이전트가 실행 중인 기기에서는 항상 에이전트 인증이 먼저 시도되어 이 알림이 거의 나타나지 않았습니다
- **상태 점검이 대화형 연결 시 저장된 키 암호를 재사용하게 되었습니다**: 이 암호는 이전에는 일반 연결에서만 적용되고 상태 점검에서는 전혀 조회되지 않아서, 터미널에서는 문제없이 동작하는 암호로 보호된 키가 상태 점검에서는 항상 실패했습니다

## [0.5.23] - 2026-07-22

### 수정
- **시작 시 테마 스크립트가 한 번도 실행되지 않았습니다**: 화면을 그리기 전에 저장된 테마·강조색·언어를 적용하는 역할이지만, 인라인으로 작성되어 CSP가 실행을 거부했고 그 결과 시작할 때 잘못된 색이 잠깐 보였습니다. 보안 정책을 완화하지 않고 별도 파일로 옮겼습니다
- **frame-ancestors를 응답 헤더로 전달**: 이 지시문은 `<meta>` CSP에서 무시되어 아무 효과가 없었습니다. 이제 app:// 응답 헤더와 개발 서버에서 전달하며, 인라인 스크립트가 다시 생기면 실패하는 테스트를 추가했습니다

### 기능
- **cast 녹화 재생**: asciinema cast v2를 기록할 수는 있어도 열 수는 없었습니다. 재생·일시정지·탐색·1x/2x/4x를 지원하며, 중간에 끊긴 녹화는 손상된 줄을 건너뛰고 개수를 알려 줍니다(파일 전체를 거부하지 않음)
- **세션 로그 내 검색**: 로그 뷰어에서 Cmd/Ctrl+F. 실시간 터미널의 검색과 서로 간섭하지 않습니다
- **Hex 패널의 행당 바이트 수**: 8 / 16 / 32 전환, 이미 수집된 출력이 즉시 다시 배치됩니다
- **변경 내역 분류 필터**: 보안 / 기능 / 수정 / 개선 칩과 항목 수 표시

### 개선
- **도달할 수 없던 선언 정리**: 정의만 되고 한 번도 호출되지 않은 코드를 제거했습니다. 검증하지 않으면서 존재를 주장하던 팀 권한, 설정해도 적용되지 않던 그룹 필드, 메인 프로세스 구현과 중복되어 결코 로드될 수 없던 WAN 초대 파서 등입니다
- **북마크 기준점 테스트 보강**: 바이트 오프셋과 줄 번호 변환을 CRLF·범위 초과·정확한 왕복 경계에서 고정했습니다

## [0.5.22] - 2026-07-21

### 보안
- **팀 감사 서명을 실제로 검증**: 이전에는 이벤트에 `sig` 필드만 있으면 확인 표시를 그렸을 뿐 검증하지 않았습니다. 이제 각 항목을 HMAC로 검증하고 검증됨 / 변조됨 / 서명 없음 / 검증 불가 네 가지 상태로 구분합니다
- **프로브가 자격 증명을 되돌려 보내지 않습니다**: 프록시 연결 테스트는 원본 메시지 대신 오류 코드만 반환합니다(HTTP CONNECT는 `Proxy-Authorization`을, ProxyCommand는 명령줄을 그대로 노출하기 때문). 데이터 소스 필드 매핑은 인벤토리의 비밀 검사를 우회할 수 없습니다

### 기능
- **SFTP 일괄 이름 변경**: `{name}` / `{ext}` / `{n}` 템플릿과 자릿수 채움을 지원하며, 실행 전에 전체 계획을 세워 미리 보여 줍니다. 이름이 겹치거나 변경 대상이 아닌 파일과 충돌하면 전체 작업을 거부합니다
- **호스트 필드 일괄 편집**: 선택한 호스트의 사용자 이름, 그룹, 포트, 태그를 한 번에 변경합니다. 태그는 대체가 아니라 추가되며, 빈 값은 기존 값 유지를 뜻하고, 데이터 소스가 관리하는 호스트는 개수와 함께 제외됩니다
- **구조화된 호스트 검색**: `tag:` `user:` `group:` `host:` 필터를 자유 텍스트와 함께 쓸 수 있습니다. 콜론 앞이 알려진 필드 이름일 때만 필터로 취급하므로 IPv6 주소나 전각 콜론이 들어간 레이블은 영향을 받지 않습니다
- **F1 단축키 목록**: 검색 가능하고 분류별로 묶이며, 실제로 적용 중인 키 바인딩을 읽습니다. 이 항목 자체도 재지정 가능한 일반 단축키입니다
- **시리얼 16진수 전송**: 문자셋 인코더를 우회해 입력한 그대로 바이트를 장치에 씁니다. 바이트 미리보기와 ASCII 표시를 함께 제공합니다
- **데이터 소스 필드 매핑**: `name` / `ip` / `ssh_port` 같은 이름을 표준 필드에 대응시켜, 수정할 수 없는 외부 인벤토리도 사용할 수 있습니다
- **기존 기능으로 가는 경로**: tmux 창으로 명령 보내기, 프록시 연결 테스트, 키 가져올 때 인증서 인식, PKCS#11 모듈 경로 추천

### 수정
- **팀 Vault 가져오기가 더 이상 거짓 성공을 알리지 않습니다**: 공유 패키지를 가져오면 "호스트 N대를 가져왔습니다"라고 알리면서도 호스트를 버려 목록이 그대로였습니다. 이제 실제로 Vault에 기록하고 실제로 추가된 개수를 보고하며, 추가만 할 뿐 로컬 변경이나 자격 증명을 덮어쓰지 않습니다
- **WAN 초대 코드로 참여할 수 있습니다**: UI가 받아들인다고 명시했는데도 `magies-follow:2:` 초대는 항상 LAN으로 해석되어 `version` 오류가 났습니다. 이제 초대 버전으로 전송 방식을 고릅니다
- **호스트 내보내기가 선택을 따릅니다**: 일부만 선택해도 전체가 내보내졌습니다
- **긴 메뉴 레이블이 더 이상 넘치지 않습니다**: 고정 너비 때문에 가장 긴 항목이 메뉴 밖으로 밀려나고 아이콘이 0폭으로 찌그러졌습니다. 정렬 메뉴는 프랑스어에서 세 항목이 넘쳤습니다
- **tmux 명령 보내기를 쓸 수 있습니다**: UI가 제공할 수 없는 창 번호를 요구해, 기능은 있어도 실행할 수 없었습니다

### 개선
- **SFTP 전송에 남은 시간 표시**: 속도를 알 수 없으면 자리를 차지하지 않고, 총량을 파일 수로 세는 디렉터리 행에서는 표시하지 않습니다
- **상단 바에 동기화 신선도 표시**: 패널을 열지 않아도 데이터가 마지막으로 실제 반영된 시점을 확인할 수 있습니다
- **AI 대화 내보내기가 추론 과정을 유지**: Markdown에서는 접어서, 일반 텍스트에서는 들여써서 소요 시간과 함께 남깁니다
- **AI 설정 사용성**: 빠른 메시지에 검색을 추가했고, 로컬 모델에서 도구 호출이 감지되지 않을 때 그 의미를 설명합니다
- **크래시 보고의 전송량 표시**: 업로드에 성공한 경우에만 세므로 실패나 중복은 포함되지 않습니다
- **프로세스와 tmux 작업에 앱 내 대화상자 사용**: 시스템 기본 창을 쓰지 않고 앱 테마를 따릅니다
- **모바일을 품질 게이트에 포함**: `mobile/`은 lint와 테스트에서 완전히 제외되어 있었고, 그 테스트는 CI에서 한 번도 실행된 적이 없었습니다

### 보안

### 참고 사항
- `checkNow` 의미: GitHub API(`performCheck`)로 새 버전을 감지, 업데이트가 있고 electron-updater가 아직 다운로드를 시작하지 않았다면 비동기로 `bridge.checkForUpdate()`를 트리거해 자동 다운로드 흐름 시작
- 이 기능은 패키지된 앱(Windows NSIS, macOS dmg/zip, Linux AppImage)에서만 유효, dev 모드는 `forceDevUpdateConfig=true` + `dev-app-update.yml` 테스트 필요(`.gitignore` 참조)
- 이전 `hasUpdate` 토스트는 `autoDownloadStatus !== 'idle'` 동안 자동 억제, 새 토스트와 중복 방지

### CI / 빌드 개선
- macOS / Windows 빌드 건너뛰기(유료 코드 서명 인증서 필요), 무료 Linux 배포 패키지 제공에 집중
- Linux x64(AlmaLinux 8) 컴파일러 업그레이드: Clang 우선, gcc-toolset-13 폴백
- Linux arm64(Debian Bullseye) 컴파일러 업그레이드: `build-essential`에서 `clang-14 + lld-14`로
- Release 작업이 macOS/Windows 빌드에 의존하지 않고, tag 푸시 후 Linux 산출물 기반으로 직접 Release 게시
- deb 산출물 검증 완화: 파일을 찾지 못하면 error 대신 warning 출력, 플랫폼 건너뛰기로 인한 CI 실패 방지
