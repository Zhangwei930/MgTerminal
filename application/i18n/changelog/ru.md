# Журнал изменений


## [0.5.27] - 2026-07-23

### Возможности
- **Лёгкий клиент баз данных**: в хранилище появился тип подключения к базе данных — можно подключаться к MySQL, PostgreSQL, SQL Server и Oracle через уже сохранённый SSH-туннель хоста, без необходимости открывать порт базы данных на самом сервере; открытие подключения к базе данных открывает новую вкладку так же, как сессия терминала, со встроенным SQL-редактором и таблицей результатов запроса; пароли баз данных используют то же шифрование хранилища, что и пароли хостов, вместо хранения в открытом виде в локальном хранилище

## [0.5.26] - 2026-07-23

### Возможности
- **Добавление хоста из контекстного меню группы**: в контекстном меню группы в дереве хостов теперь есть «Новый хост», который сразу указывает группу нового хоста вместо создания через верхнюю кнопку с ручным выбором группы

### Улучшения
- **Питомец на рабочем столе в режиме ожидания теперь шагает, а не дышит на месте**: анимация ожидания заставляет питомца шагать влево-вправо внутри плавающего окна, с пружинящим шагом и разворотом на краю; у стандартного персонажа нет спрайт-листа для ходьбы, поэтому движение — чистая иллюзия на CSS-трансформациях
- **Реал-тайм график монитора переоформлен в неоновую HUD-панель**: график во вкладке обзора сменил плавную линию на ступенчатую, на фиксированном тёмном фоне со звёздами, с более сильным двухслойным свечением и расширяющимся радар-пинг-кольцом на последнем значении каждой серии

## [0.5.25] - 2026-07-22

### Возможности
- **Питомец на рабочем столе**: включите в Настройки → ИИ → Питомец, и перетаскиваемый плавающий питомец появится в любом месте экрана, анимируясь в соответствии со статусом ИИ — дыхание в покое, подпрыгивание во время работы, покачивание в ожидании вашего подтверждения, взмах при завершении, дрожь при ошибке. Поскольку питомец редко получает системный фокус, для него отдельно отключено ограничение анимации Electron для неактивных окон, чтобы он не выглядел замершим
- **Прямые действия по клику**: клик открывает/фокусирует панель чата ИИ, по возможности переходя к занятой сессии терминала; двойной клик фокусирует главное окно; правый клик открывает меню для запуска команды, заданной в настройках, открытия настроек ИИ, сброса позиции питомца или его скрытия; при наведении показывается более подробный пузырь статуса
- **Настраиваемый внешний вид**: можно загрузить собственное изображение или спрайт-лист, с диапазонами кадров по статусам (ожидание/выполнение/ожидание подтверждения/готово/ошибка) для спрайт-листов; размер, непрозрачность, режим поверх всех окон и видимость пузыря настраиваются
- **Режим приватности и уведомления о завершении**: режим приватности ограничивает пузырь общим статусом вроде «выполняется» вместо названия активного инструмента; задачи длительностью от 10 секунд могут по желанию вызывать системное уведомление при завершении или ошибке
- **Сохранение позиции и поддержка нескольких мониторов**: питомец запоминает, куда вы его перетащили, даже после перезапуска или повторного включения; он автоматически возвращается в угол по умолчанию, если монитор отключается или меняется разрешение

## [0.5.24] - 2026-07-22

### Исправления
- **Проверка состояния так и не читала файл ключа**: асинхронная функция чтения приватного ключа вызывалась без `await`, поэтому проверялось не содержимое файла, а неразрешённый Promise, и каждый раз ключ признавался «не похожим на приватный» и молча пропускался. Любой хост, полагающийся на локальный файл ключа, а не на встроенно сохранённый ключ, гарантированно проваливал проверку состояния, хотя то же самое соединение в терминале работало нормально
- **Локальная ошибка расшифровки больше не выдаётся за отказ во входе**: пароль или ключ, всё ещё остающийся зашифрованным placeholder'ом, обнулялся перед проверкой, и сервер закономерно отклонял вход без каких-либо учётных данных. Теперь проверка распознаёт ситуацию «учётные данные заданы, но это устройство не может их расшифровать» и указывает на разблокировку хранилища или восстановление защищённого хранилища
- **Недоверенный ключ хоста больше не маскируется под ошибку аутентификации**: проверка и раньше отклоняла все методы аутентификации, если ключ хоста неизвестен или изменился, но никогда не сообщала об этом факте панели. Теперь показывается отдельный статус «ключ хоста не проверен» с предложением подключиться один раз вручную, чтобы установить доверие
- **Уведомление «зашифрованный ключ пропущен» больше не зависит от случая**: раньше оно появлялось только тогда, когда вообще не было опробовано ни одного метода аутентификации, но на любой машине с запущенным SSH-агентом сначала всегда пробуется агент — из-за этого уведомление почти никогда не срабатывало
- **Проверка состояния теперь использует пароль ключа, сохранённый при интерактивном подключении**: раньше этот пароль применялся только при обычном подключении и никогда не запрашивался проверкой состояния, поэтому защищённый паролем ключ, прекрасно работающий в терминале, всегда проваливал проверку состояния

## [0.5.23] - 2026-07-22

### Исправления
- **Скрипт темы при запуске никогда не выполнялся**: он применяет сохранённые тему, акцент и язык до отрисовки интерфейса, но как встроенный блок был отклонён CSP, из-за чего при запуске мелькали неверные цвета. Вынесен в отдельный файл без послабления политики безопасности
- **frame-ancestors теперь приходит заголовком**: в `<meta>` эта директива игнорируется и ничего не делала. Теперь её отдают заголовки app:// и сервер разработки, а новый тест падает при появлении встроенного скрипта

### Возможности
- **Воспроизведение записей cast**: приложение умело записывать asciinema cast v2, но не открывать. Есть воспроизведение, пауза, перемотка и 1x/2x/4x; оборванная запись пропускает повреждённые строки и показывает их число, а не отклоняется целиком
- **Поиск внутри журнала сеанса**: Cmd/Ctrl+F в просмотрщике журналов, независимо от поиска в живом терминале
- **Байтов в строке hex-панели**: переключение 8 / 16 / 32 с мгновенной перекомпоновкой уже собранного вывода
- **Фильтр списка изменений по категориям**: чипы «безопасность / возможности / исправления / улучшения» с числом пунктов

### Улучшения
- **Удалены объявления, которые не могли сработать**: код, определённый и ни разу не вызванный, включая командное право, которое никогда не проверялось, но заявлялось, поле группы без эффекта при установке и парсер WAN-приглашений, дублирующий реализацию главного процесса и не подлежащий загрузке
- **Покрытие опорной точки закладок**: перевод байтового смещения в номер строки зафиксирован на CRLF, выходе за диапазон и точном обратном преобразовании

## [0.5.22] - 2026-07-21

### Безопасность
- **Подписи аудита команды действительно проверяются**: раньше галочка ставилась при наличии поля `sig` без какой-либо проверки; теперь каждая запись проверяется по HMAC и получает одно из четырёх состояний — проверено, изменено, без подписи, проверить нечем
- **Проверки не возвращают учётные данные**: тест подключения через прокси отдаёт код ошибки, а не исходное сообщение (HTTP CONNECT отражает `Proxy-Authorization`, ProxyCommand — командную строку); сопоставление полей источника не может обойти проверку секретов в инвентаре

### Возможности
- **Массовое переименование в SFTP**: шаблоны `{name}` / `{ext}` / `{n}` с дополнением нулями; весь набор планируется и показывается до запуска, а любое совпадение имён или столкновение с посторонним файлом отменяет всю операцию
- **Массовое изменение полей хостов**: имя пользователя, группа, порт и теги сразу для всей выборки; теги добавляются, а не заменяют список, пустое поле означает «оставить как есть», а хосты под управлением источника исключаются с указанием количества
- **Структурированный поиск хостов**: фильтры `tag:` `user:` `group:` `host:` вместе со свободным текстом; термин считается фильтром только если перед двоеточием стоит известное имя поля, поэтому адреса IPv6 и метки с полноширинным двоеточием не затрагиваются
- **Список горячих клавиш по F1**: с поиском, сгруппированный по категориям и читающий действующие привязки; сам он тоже обычное переназначаемое сочетание
- **Отправка HEX в последовательный порт**: байты записываются в устройство как введены, минуя кодировщик кодировки, с предпросмотром байтов и ASCII
- **Сопоставление полей источника данных**: имена вроде `name` / `ip` / `ssh_port` переносятся на канонические поля, что позволяет использовать чужие инвентари без правки
- **Доступ к уже готовым возможностям**: отправка команды в окно tmux, проверка подключения через прокси, распознавание сертификата при импорте ключа и подсказки путей к модулям PKCS#11

### Исправления
- **Импорт командного Vault больше не сообщает о ложном успехе**: импорт пакета объявлял «импортировано N хостов», отбрасывая их, и список не менялся; теперь хосты действительно попадают в Vault, сообщается фактическое количество, и они добавляются, не затирая локальные правки и учётные данные
- **По коду приглашения WAN можно подключиться**: приглашения `magies-follow:2:` всегда разбирались как LAN и завершались ошибкой `version`, хотя интерфейс прямо предлагал их принимать; теперь транспорт выбирается по версии приглашения
- **Экспорт хостов учитывает выбор**: после выделения части хостов экспортировались все
- **Длинные пункты меню больше не выходят за края**: из-за фиксированной ширины самые длинные строки выходили за пределы меню, а их значки сжимались до нуля; в меню сортировки на французском переполнялись три пункта
- **Отправка команды в tmux работает**: действие требовало номер панели, который интерфейс не мог предоставить, поэтому функция была, но вызвать её было нельзя

### Улучшения
- **В передачах SFTP показано оставшееся время**: отсутствует, когда скорость неизвестна, и не выводится для строк каталогов, где итог считается в файлах
- **Свежесть синхронизации в верхней панели**: при наведении видно, когда данные в последний раз действительно были записаны, без открытия панели
- **Экспорт диалогов ИИ сохраняет рассуждения**: свёрнуто в Markdown, с отступом в обычном тексте, с указанием длительности
- **Удобство настроек ИИ**: в быстрых сообщениях появился поиск, а при отсутствии вызова инструментов у локальной модели поясняется, что это значит
- **Отчёты о сбоях показывают отправленное**: учитывается только после успешной загрузки, поэтому неудачи и повторы не увеличивают счётчик
- **Действия с процессами и tmux используют диалог приложения**: без системных окон, подтверждение следует теме приложения
- **Мобильная часть под контролем качества**: каталог `mobile/` был полностью исключён из lint и тестов, а его тест ни разу не выполнялся в CI

## [0.5.21] - 2026-07-21

### Безопасность
- **Глубокая очистка контекста хоста**: пароли, Telnet-пароли, закрытые ключи и кодовые фразы рекурсивно удаляются из вложенных объектов и массивов
- **Область сессии публичных возможностей**: каждая публичная возможность использует fail-closed проверку и не выходит за пределы терминальных сессий текущего чата
- **Чувствительное чтение и аудит подтверждений**: Pod describe стал Sensitive Read; главный процесс сохраняет только метаданные подтверждений без аргументов и учётных данных

### Функции
- **Строгая локальная приватность и проверка модели**: только loopback-модели, без внешних Agent/веб-поиска, с проверкой вызова инструментов
- **Управление Docker Compose**: просмотр проектов и сервисов, явные операции up, restart и down
- **Расширенные операции Kubernetes**: структурированные Events, статус/история/перезапуск rollout, Agent Exec, интерактивный Exec и loopback Port Forward

### Исправления
- **JSON-разбор списков Kubernetes**: Namespaces, Pods и Deployments больше не зависят от нестабильных таблиц, ошибки kubectl показываются напрямую
- **Кроссплатформенные пакеты**: исправлен сбой Apple Silicon после Fuses без платного Developer ID; Android использует JDK 21, SDK/Build Tools 36 и метаданные v0.5.21

## [0.2.0] - 2026-07-11

### Функции
- Исправлена отправка IPC-событий автообновления только одному окну; теперь они транслируются всем окнам (главное + настройки получают оба)
- Унифицированы конечные автоматы ручной проверки и автообновления, устранены три параллельных состояния
- Ручная «Проверка обновлений» определяет версии через GitHub API и при находке асинхронно запускает загрузку electron-updater
- После клика «Проверить обновления» в окне настроек прогресс загрузки отражается в интерфейсе в реальном времени
- Приложение автоматически запускает проверку `electron-updater` через 5 секунд после старта, без ручного клика
- При обнаружении новой версии загрузка начинается автоматически (`autoDownload=true`)
- По завершении загрузки появляется постоянный тост; клик «Перезапустить сейчас» устанавливает обновление
- При сбое загрузки появляется тост ошибки с запасным пунктом «Открыть Releases»
- Полоса прогресса в Settings > System показывает автозагрузку в реальном времени, единообразно управляясь `useUpdateCheck`
- Платформы Linux deb/rpm/snap и другие без поддержки electron-updater пропускаются автоматически, сохраняя прежнее поведение уведомлений через GitHub API

### Заметки о дизайне
- `broadcastToAllWindows` заменяет одиночную отправку `getSenderWindow`, гарантируя получение IPC-событий всеми окнами
- Поле `manualCheckStatus` отслеживает состояние интерфейса ручной проверки (idle/checking/available/up-to-date/error) и отрисовывается вместе с `autoDownloadStatus` по приоритету
- `SettingsSystemTab` больше не держит локальное состояние обновления; он однонаправленно получает единые данные `useUpdateCheck`
- Две прежде независимые системы (уведомления GitHub API + ручная загрузка electron-updater) объединены в один конечный автомат: `useUpdateCheck` — единственный источник истины, управляющий и тостом `App.tsx`, и полосой прогресса `SettingsSystemTab`
- Глобальные постоянные IPC-слушатели регистрируются один раз в `autoUpdateBridge.init()`, избегая повторной регистрации/очистки при каждом ручном запросе загрузки
- `autoInstallOnAppQuit=false`: без тихой установки, перезапуск инициирует пользователь

### Изменения интерфейса（SettingsSystemTabProps）
- Удалены: `autoDownloadStatus`, `downloadPercent`
- Добавлены: `updateState` (полный UpdateState), `checkNow`, `installUpdate`, `openReleasePage`

### Примечания
- Семантика `checkNow`: определяет новые версии через GitHub API (`performCheck`); если обновление есть и electron-updater ещё не начал загрузку, асинхронно вызывается `bridge.checkForUpdate()` для запуска автозагрузки
- Функция работает только в упакованных приложениях (Windows NSIS, macOS dmg/zip, Linux AppImage); режим dev требует `forceDevUpdateConfig=true` + `dev-app-update.yml` для тестов (см. `.gitignore`)
- Старый тост `hasUpdate` подавляется, пока `autoDownloadStatus !== 'idle'`, исключая дублирование с новым тостом

### Улучшения CI / сборки
- Пропущены сборки macOS / Windows (нужны платные сертификаты подписи кода) в пользу бесплатных пакетов Linux
- Обновление компилятора Linux x64 (AlmaLinux 8): предпочтителен Clang, откат на gcc-toolset-13
- Обновление компилятора Linux arm64 (Debian Bullseye): с `build-essential` на `clang-14 + lld-14`
- Задача релиза больше не зависит от сборок macOS/Windows; пуш тега публикует релиз напрямую из артефактов Linux
- Смягчена проверка артефактов deb: отсутствующие файлы дают предупреждение вместо ошибки, чтобы пропуски платформ не валили CI
