# 更新履歴


## [0.5.23] - 2026-07-22

### 修正
- **起動時のテーマ適用スクリプトが一度も実行されていませんでした**：描画前に保存済みのテーマ・アクセント・言語を適用する役割ですが、インライン記述のため CSP に実行を拒否され、起動時に誤った配色が一瞬表示されていました。セキュリティポリシーを緩めずに独立ファイルへ移動しました
- **frame-ancestors をレスポンスヘッダーで配信**：この指示は `<meta>` の CSP では無視され、実質無効でした。app:// のレスポンスヘッダーと開発サーバーから配信するようにし、インラインスクリプトの再発を検出するテストを追加しました

### 新機能
- **cast 録画の再生**：asciinema cast v2 を記録できるのに開けませんでした。再生・一時停止・シーク・1x/2x/4x に対応し、途中で切れた録画は破損行を飛ばして件数を示します（ファイル全体を拒否しません）
- **セッションログ内の検索**：ログビューアで Cmd/Ctrl+F。ライブ端末側の検索とは独立して動作します
- **Hex パネルの 1 行あたりバイト数**：8 / 16 / 32 を切り替え、取得済みの内容が即座に再配置されます
- **変更履歴のカテゴリ絞り込み**：セキュリティ / 新機能 / 修正 / 改善のチップと件数表示

### 改善
- **到達し得ない宣言の整理**：定義されたまま一度も呼ばれないコードを削除しました。検証されないのに存在を主張していたチーム権限、設定しても効かないグループ項目、メインプロセス側と重複し読み込まれ得ない WAN 招待の解析モジュールなどです
- **ブックマーク基点のテスト強化**：バイトオフセットと行番号の相互変換について CRLF・範囲外・完全往復の境界を網羅しました

## [0.5.22] - 2026-07-21

### セキュリティ
- **チーム監査署名を実際に検証**：これまでは `sig` フィールドがあるだけでチェックマークを表示し、検証は一切していませんでした。各エントリを HMAC で検証し、検証済み / 改ざん / 未署名 / 検証不能の 4 状態を区別します
- **プローブが資格情報を持ち帰らない**：プロキシ接続テストは元のメッセージではなくエラーコードのみを返します（HTTP CONNECT は `Proxy-Authorization` を、ProxyCommand はコマンドラインを反映するため）。データソースのフィールドマッピングはインベントリの機密チェックを回避できません

### 新機能
- **SFTP の一括リネーム**：`{name}` / `{ext}` / `{n}` テンプレートとゼロ埋めに対応し、実行前に全体の計画を作成してプレビューします。名前の重複やリネーム対象外のファイルとの衝突があればバッチ全体を拒否します
- **ホストのフィールド一括編集**：選択したホストのユーザー名、グループ、ポート、タグをまとめて変更します。タグは置換ではなく追加、空欄は現状維持、データソース管理のホストは件数を示して除外されます
- **構造化ホスト検索**：`tag:` `user:` `group:` `host:` フィルタとフリーテキストの併用に対応。コロンの前が既知のフィールド名の場合のみフィルタとして扱うため、IPv6 アドレスや全角コロンを含むラベルは影響を受けません
- **F1 キーボードショートカット一覧**：検索可能でカテゴリ別に整理され、実際に有効な割り当てを読み取ります。これ自体も再割り当て可能な通常のショートカットです
- **シリアル 16 進送信**：文字セットエンコーダを迂回して入力どおりのバイトをデバイスへ書き込み、バイトプレビューと ASCII 表示を添えます
- **データソースのフィールドマッピング**：`name` / `ip` / `ssh_port` のような名前を正規フィールドへ対応付け、編集できない外部インベントリでも利用できます
- **既存機能への導線**：tmux ウィンドウへのコマンド送信、プロキシ接続テスト、鍵インポート時の証明書認識、PKCS#11 モジュールパスの候補

### 修正
- **チーム Vault のインポートが偽の成功を報告しなくなりました**：共有パッケージのインポートは「N 台のホストをインポートしました」と表示しながらホストを破棄し、一覧は変化しませんでした。実際に Vault へ書き込み、実際に追加された件数を報告します。追加のみでローカルの変更や資格情報を上書きしません
- **WAN 招待コードで参加できます**：UI が受け付けると明記しているにもかかわらず、`magies-follow:2:` の招待は常に LAN として解析され `version` エラーになっていました。招待のバージョンで転送方式を選択します
- **ホストのエクスポートが選択を尊重**：一部を選択してエクスポートしても全件が出力されていました
- **長いメニュー項目がはみ出さなくなりました**：固定幅のため最長の項目がメニュー外へあふれ、アイコンが幅ゼロに潰れていました。ソートメニューはフランス語で 3 項目があふれていました
- **tmux のコマンド送信が使えます**：UI が提供できないペイン番号を要求していたため、機能は存在しても実行できませんでした

### 改善
- **SFTP 転送に残り時間を表示**：速度が不明なときは場所を取らず、総量をファイル数で数えるディレクトリ行では表示しません
- **トップバーに同期の鮮度を表示**：パネルを開かなくても、データが最後に実際に反映された時刻をホバーで確認できます
- **AI 会話のエクスポートが思考過程を保持**：Markdown では折りたたみ、プレーンテキストではインデントし、所要時間を添えます
- **AI 設定の使い勝手**：クイックメッセージに検索を追加。ローカルモデルでツール呼び出しが検出されない場合、その意味を説明します
- **クラッシュレポートの送信数を表示**：アップロード成功時のみ加算し、失敗や重複は含めません
- **プロセスと tmux の操作をアプリ内ダイアログに統一**：OS 標準のダイアログを使わず、アプリのテーマに従います
- **モバイルを品質ゲートの対象に**：`mobile/` は lint とテストから完全に除外され、そのテストは CI で一度も実行されていませんでした
