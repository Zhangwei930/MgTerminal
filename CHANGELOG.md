# Changelog


## [0.5.25] - 2026-07-22

### 功能
- **桌面宠物**：在 设置 → AI → Pet 中开启后，一只可拖动的悬浮宠物会出现在屏幕任意角落，跟随 AI 状态切换动画——空闲呼吸、运行中跳动、等待你确认时摇晃、完成时挥手、失败时抖动；宠物本就很少获得系统焦点，因此专门关闭了 Electron 对未聚焦窗口的动画限流，避免看起来像"死"的
- **点击直达**：单击宠物打开/聚焦 AI 对话面板，并尽量跳转到当前正忙的那个终端会话标签；双击聚焦主窗口；右键弹出菜单，可运行你在设置里配置的一条自定义命令、打开 AI 设置、重置宠物位置或隐藏宠物；悬停显示更详细的状态气泡
- **自定义外观**：可上传自己的图片或精灵图作为宠物，精灵图支持按状态（空闲/运行/等待/完成/失败）分别指定播放的帧区间；大小、透明度、是否始终置顶、是否显示气泡均可调节
- **隐私模式与完成提醒**：隐私模式下气泡只显示"运行中"这类通用状态，不透出具体在跑哪个工具；耗时 10 秒以上的任务完成或失败时，可选弹出系统桌面通知
- **位置持久化与多屏支持**：拖动后的位置会记住，重启应用或重新开启宠物后自动恢复；显示器断开或分辨率变化导致宠物移出可视区域时自动归位

## [0.5.24] - 2026-07-22

### 修复
- **健康检查探测器此前从未真正读取过密钥文件**：读取私钥的异步函数遗漏了 await，返回的是未展开的 Promise 而非文件内容，因此每次都被判定为「不像私钥」并静默跳过。所有依赖本地密钥文件而非内联存储的主机，在健康检查里必然认证失败，即使终端连接完全正常
- **本机无法解密的凭据不再被误判为认证被拒**：仍是加密占位符的密码或私钥会被直接剥离为空后再探测，服务器自然会拒绝零凭据的登录；现在会先识别出「配置了凭据但本机解不开」，并提示解锁保险库或修复安全存储
- **主机密钥尚未受信任时不再误报为认证失败**：探测器早已在主机密钥未知或变更时主动扣留所有认证方式，却从未把这一事实回传给面板；现在会显示专门的「主机密钥未验证」状态，并提示先手动连接一次以确认信任
- **加密密钥被跳过的提示不再依赖运气**：此前只有在没有尝试过任何认证方式时才会显示「密钥已加密且未保存密码」，但只要本机有 SSH agent 在运行就必然会先尝试 agent 认证，导致这条提示几乎从不出现
- **健康检查现在会复用交互式连接时已保存的密钥密码**：此前该密码只在正常连接里生效，健康检查从不查询，导致带密码保护的私钥在终端里能用、放进健康检查却总是报认证失败

## [0.5.23] - 2026-07-22

### 修复
- **启动主题脚本此前从未执行**：它负责在界面渲染前套用已保存的主题、强调色与语言，但因内联写法被 CSP 直接拒绝运行，导致启动瞬间会闪出错误配色。现已移出为独立文件，未放宽任何安全策略
- **frame-ancestors 改由响应头下发**：该指令写在 `<meta>` 里会被浏览器忽略、实际不生效；现由 app:// 响应头与开发服务器下发，并新增测试防止内联脚本再次出现

### 功能
- **Cast 录制回放**：应用此前能录制 asciinema cast v2 却打不开，现支持播放/暂停/进度跳转/1x·2x·4x 倍速；被中途截断的录制会跳过损坏行并提示条数，而不是拒绝整个文件
- **会话日志内搜索**：日志查看器支持 Cmd/Ctrl+F 检索，与实时终端的搜索互不干扰
- **Hex 面板每行字节数**：可在 8 / 16 / 32 间切换，切换后已捕获内容立即重排
- **更新日志按类别筛选**：安全 / 功能 / 修复 / 优化分类芯片，各显条目数

### 优化
- **清理无法生效的声明**：移除了一批定义后从未被调用的代码，包括一个从不校验却声称存在的团队权限、一个设了永不生效的分组字段，以及一份与主进程实现重复、永远不会被加载的 WAN 邀请解析模块
- **书签锚点测试加固**：补齐字节偏移与行号互转在 CRLF、越界、精确往返等边界上的覆盖

## [0.5.22] - 2026-07-21

### 安全
- **团队审计签名真实验签**：此前只要事件带 `sig` 字段就画对勾，从不校验；现在逐条 HMAC 验签，并区分「已验证 / 被篡改 / 未签名 / 无密钥可验」四种状态
- **探测不再回传凭据**：代理连接测试只返回错误码而非原始报错（HTTP CONNECT 会回显 `Proxy-Authorization`，ProxyCommand 会回显命令行）；数据源字段映射无法绕过清单的密钥检查

### 功能
- **SFTP 批量重命名**：支持 `{name}` / `{ext}` / `{n}` 模板与序号补零，执行前算出整批计划并逐条预览；任一处重名或撞上未参与重命名的文件即整批拒绝
- **主机字段批量编辑**：一次修改选中主机的用户名、分组、端口与标签；标签为追加而非替换，留空表示保持原值；数据源托管的主机被排除并提示条数
- **结构化主机搜索**：支持 `tag:` `user:` `group:` `host:` 过滤与自由文本混用；仅在冒号前是已知字段名时才视作过滤器，IPv6 地址与含全角冒号的中文标签不受影响
- **F1 快捷键速查表**：可搜索、按分类分组，读取当前实际生效的按键绑定；本身也是一条可改键的普通快捷键
- **串口十六进制发送**：按原样写入设备，绕过字符集编码层，附字节预览与 ASCII 对照
- **数据源字段映射**：可将 `name` / `ip` / `ssh_port` 之类的字段名映射到规范字段，供无法修改的外部清单使用
- **接通已有能力**：tmux 向窗口发送命令、代理连接测试、导入密钥时一并识别证书、PKCS#11 模块路径预设

### 修复
- **团队 Vault 导入不再假成功**：此前导入分享包会提示「已导入 N 台主机」，但主机被丢弃、列表毫无变化；现在真正写入 Vault 并报告实际落地数量，且只增不覆盖本地改动与凭据
- **WAN 邀请码可以加入了**：`magies-follow:2:` 邀请一律被按 LAN 解析并报 `version` 错误，尽管界面明写支持；现在按邀请版本分流传输层
- **主机导出遵守勾选**：勾选部分主机后导出仍会导出全部
- **下拉菜单长标签不再溢出**：固定宽度导致最长的条目文字跑出菜单、图标被挤成零宽；排序菜单在法语下有三个选项溢出
- **tmux 发送命令可用**：该操作要求一个界面无法提供的窗格序号，导致功能存在却无法触发

### 优化
- **SFTP 传输显示剩余时间**：速率未知时不占位；目录行不显示（其总量按文件数计）
- **顶栏显示同步新鲜度**：悬停即可看到数据上次真正落地的时间，无需点开面板
- **AI 对话导出保留思考过程**：Markdown 折叠展示，纯文本缩进展示，并带耗时
- **AI 设置体验**：快捷消息支持搜索；本地模型未检测到工具调用时说明其影响
- **崩溃报告可见发送量**：仅在上传成功后计数，失败与去重不计入
- **进程与 tmux 操作改用应用内确认框**：不再使用系统原生弹窗，跟随应用主题
- **移动端纳入质量门禁**：`mobile/` 此前被 lint 与测试完全排除，其测试从未被 CI 执行过

### 设计原理
- `broadcastToAllWindows` 替换 `getSenderWindow` 单点发送，保证所有窗口都能收到 IPC 事件
- `manualCheckStatus` 字段追踪手动检查 UI 状态（idle/checking/available/up-to-date/error），与 `autoDownloadStatus` 在 UI 层按优先级渲染
- `SettingsSystemTab` 不再持有本地 update state，单向接收 `useUpdateCheck` 统一数据
- 将原有两套独立系统（GitHub API 通知 + electron-updater 手动下载）合并为统一状态机：`useUpdateCheck` 作为唯一事实来源，同时驱动 `App.tsx` toast 和 `SettingsSystemTab` 进度条
- 全局持久化 IPC 监听器在 `autoUpdateBridge.init()` 时一次性注册，避免每次手动下载请求重复注册/清理监听器
- `autoInstallOnAppQuit=false`，不做静默安装，由用户主动触发重启

### 接口变更（SettingsSystemTabProps）
- 移除：`autoDownloadStatus`、`downloadPercent`
- 新增：`updateState`（完整 UpdateState）、`checkNow`、`installUpdate`、`openReleasePage`

### 注意事项
- `checkNow` 语义：使用 GitHub API（`performCheck`）检测是否有新版本，若发现更新且 electron-updater 尚未开始下载，则异步触发 `bridge.checkForUpdate()` 启动自动下载流程
- 此功能仅对打包后的应用（Windows NSIS、macOS dmg/zip、Linux AppImage）生效，dev 模式需配合 `forceDevUpdateConfig=true` + `dev-app-update.yml` 测试（见 `.gitignore`）
- `hasUpdate` 旧 toast 在 `autoDownloadStatus !== 'idle'` 时自动抑制，避免与新 toast 重复
