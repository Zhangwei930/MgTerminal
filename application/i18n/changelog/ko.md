## [0.5.9] - 2026-07-19

### 버그 수정
- **대화 상자를 닫을 수 없는 문제**: 설정 창에서 연 대화 상자(예: '업데이트 내용')가 닫히지 않던 문제를 수정했습니다. 오른쪽 위 X가 제목 표시줄 드래그 영역과 겹쳐 클릭이 창 드래그로 처리되었습니다. 이제 모든 대화 상자를 드래그 영역에서 제외하여 X로 확실히 닫을 수 있습니다

## [0.5.2] - 2026-07-18

### 기능
- **로컬 우선 팀 Vault**: 메타데이터만 포함한 호스트 인벤토리 패키지 공유, 역할(owner/editor/viewer), HMAC 서명 감사. 비밀번호와 개인 키는 기기를 떠나지 않음
- **WAN 릴레이 세션 팔로우**: NAT 친화적 공동 시청용 TCP NDJSON 릴레이. 내장 로컬 릴레이 또는 `scripts/follow-relay.cjs` 자체 호스팅
- **기기 Passkey Vault 잠금 해제**: WebAuthn 플랫폼 인증기(Touch ID / Windows Hello / 보안 키)를 메인 프로세스에서 검증. 클라우드 다중 기기 동기화 아님
- **내장 ssh2 하이브리드 양자내성 KEX**: `mlkem768x25519-sha256` 우선, 미지원 시 고전 알고리즘으로 폴백
- **RDP 호스트 지원**: Vault에서 시스템 원격 데스크톱 실행(Windows mstsc, macOS Windows App, Linux xfreerdp)
- **시스템 OpenSSH 점프/프록시**: 점프 체인과 HTTP/SOCKS 프록시를 시스템 OpenSSH 세션에서 사용

### 개선
- **전역 UI 컴포넌트 업그레이드**: 버튼, 입력, 팝오버, 사이드 패널, 빈 상태, Toast의 모서리/그림자/포커스 링 통일
- **AI 사이드바 다듬기**: Q&A 레이아웃, 모델/권한 컨트롤, 사각 생각 스피너, 입력 타이포그래피
- **변경 로그 대화상자 재설계**: 버전 접기, 섹션 색상, UI 언어 연동

## [0.5.0] - 2026-07-17

### 기능
- **터미널 Hex/Raw 스트림 진단 패널**: 선택적으로 활성화하여 세션의 원시 입출력을 바이트 단위로 확인, 인코딩/이스케이프 시퀀스 문제 분석에 유용
- **JSON 호스트 소스**: 로컬 JSON 파일 또는 HTTP(S) 엔드포인트에서 호스트 인벤토리 가져오기(CMDB / Ansible / 커스텀 API 스타일). 메타데이터 전용이며 비밀 정보가 포함된 인벤토리는 거부, HTTP 인증 헤더 지원
- **호스트 인벤토리 공유 및 가져오기**: 팀 인수인계를 위한 메타데이터 전용 인벤토리 내보내기(Ansible YAML 형식 포함), 클립보드에서 가져오기
- **이름 있는 워크스페이스 템플릿**: 호스트 바인딩, 분할 레이아웃, 선택적 cwd/시작 명령을 템플릿으로 저장하고 빠른 전환기에서 원클릭 적용
- **연결 로그 북마크**: 재생 위치 북마크 + 메모 + 검색 이동, 로그 목록에 북마크 수 표시
- **포트 포워딩 실시간 채널 보기**: 로컬/원격/동적 포워딩의 연결별 소스, 대상, 트래픽 바이트 통계
- **스크립트 onOutput 트리거 동작 확장**: 출력 패턴이 일치하면 데스크톱 알림, 알림음, 탭 마커, 세션 녹화 시작 선택 가능
- **안전 붙여넣기와 정밀 브로드캐스트**: 여러 줄 붙여넣기 지연 / 프롬프트 대기 / 위험 명령 확인. 브로드캐스트는 워크스페이스/선택/그룹/창을 정확히 지정 가능
- **시스템 OpenSSH 채널 강화**: GSSAPI/Kerberos 및 포스트 양자(PQ) 알고리즘을 시스템 OpenSSH를 통해 지원; 점프 체인 및 HTTP/SOCKS 프록시 지원
- **내장 ssh2 하이브리드 포스트 양자 KEX**: `mlkem768x25519-sha256`(ML-KEM-768 + X25519)을 우선 협상하고, 서버 미지원 시 고전 알고리즘으로 폴백; 시스템 ssh 강제 불필요
- **RDP 호스트 지원**: Vault 호스트에서 RDP를 활성화하고 시스템 원격 데스크톱 클라이언트(Windows mstsc / macOS Windows App / Linux xfreerdp) 실행
- **변경 로그가 UI 언어를 따름**: 앱 내 Changelog를 현재 UI 언어로 표시(10개 언어)

### Windows ARM64
- **win-arm64 설치 패키지에 mosh / ET 번들 추가**: MoshMagies 0.1.9와 EternalTerminal 6.2.10이 Windows arm64 네이티브 바이너리 최초 제공
- **win-arm64 전용 자동 업데이트 피드**: 업데이트 메타데이터가 전용 `latest-arm64.yml` 채널로 이동, 더 이상 x64 업데이트를 따르지 않음(이전에는 arm64 업데이트가 x64 패키지를 설치하고 에뮬레이션으로 실행됨)

## [0.4.10] - 2026-07-17

### 기능
- **SSH 연결 진단 센터**: 호스트 편집 패널의 "연결 테스트" + 연결 실패 시 "진단 실행", DNS / TCP / 점프 호스트 / 호스트 키 / 인증 / SFTP를 단계별 점검
- **SSH 에이전트 일급 인증**: 호스트별로 에이전트 인증을 명시적으로 선택, 에이전트의 키 지문 확인 및 선호 ID 지정 가능. 연결 로그에 실제 인증 방식 기록
- **다중 호스트 상태 스냅샷**: Vault에서 원클릭으로 지연 시간, 인증, 부하/메모리/디스크 일괄 점검. 비정상 호스트 필터링 후 스크립트 실행
- **SFTP 신뢰성 1단계**: 이어받기, 실패 시 자동 백오프 재시도, 전송 큐 영속화(재시작 후 복구), 선택적 SHA-256 검증
- **온보딩 개선**: 첫 빈 Vault 3단계 가이드, 빠른 전환기 명령 항목(설정/가져오기/상태 점검 등), 빈 상태 마이그레이션 힌트, 첫 연결 성공 제안, README 기능 매트릭스

### 수정
- 기존 Vault를 업그레이드한 사용자에게 첫 실행 가이드가 더 이상 표시되지 않음. 상태 점검의 인증 실패 시 점프 연결을 올바르게 종료

### 참고 사항
- `checkNow` 의미: GitHub API(`performCheck`)로 새 버전을 감지, 업데이트가 있고 electron-updater가 아직 다운로드를 시작하지 않았다면 비동기로 `bridge.checkForUpdate()`를 트리거해 자동 다운로드 흐름 시작
- 이 기능은 패키지된 앱(Windows NSIS, macOS dmg/zip, Linux AppImage)에서만 유효, dev 모드는 `forceDevUpdateConfig=true` + `dev-app-update.yml` 테스트 필요(`.gitignore` 참조)
- 이전 `hasUpdate` 토스트는 `autoDownloadStatus !== 'idle'` 동안 자동 억제, 새 토스트와 중복 방지

### CI / 빌드 개선
- macOS / Windows 빌드 건너뛰기(유료 코드 서명 인증서 필요), 무료 Linux 배포 패키지 제공에 집중
- Linux x64(AlmaLinux 8) 컴파일러 업그레이드: Clang 우선, gcc-toolset-13 폴백
- Linux arm64(Debian Bullseye) 컴파일러 업그레이드: `build-essential`에서 `clang-14 + lld-14`로
- Release 작업이 macOS/Windows 빌드에 의존하지 않고, tag 푸시 후 Linux 산출물 기반으로 직접 Release 게시
- deb 산출물 검증 완화: 파일을 찾지 못하면 error 대신 warning 출력, 플랫폼 건너뛰기로 인한 CI 실패 방지
