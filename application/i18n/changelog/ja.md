## [0.5.2] - 2026-07-18

### 機能
- **ローカル優先のチーム Vault**：メタデータのみのホスト一覧パッケージ共有、ロール（owner/editor/viewer）、HMAC 署名監査。パスワードと秘密鍵は端末外へ出しません
- **WAN 中継でのセッションフォロー**：NAT 越えの共同視聴向け TCP NDJSON 中継。内蔵ローカル中継または `scripts/follow-relay.cjs` を自前ホスティング
- **デバイス Passkey による Vault 解除**：WebAuthn プラットフォーム認証器（Touch ID / Windows Hello / セキュリティキー）をメインプロセスで検証。クラウド複数端末同期ではありません
- **内蔵 ssh2 ハイブリッド耐量子 KEX**：`mlkem768x25519-sha256` を優先し、非対応時は古典アルゴリズムへフォールバック
- **RDP ホスト対応**：Vault からシステムリモートデスクトップを起動（Windows mstsc、macOS Windows App、Linux xfreerdp）
- **システム OpenSSH のジャンプとプロキシ**：ジャンプチェーンと HTTP/SOCKS プロキシがシステム OpenSSH セッションで利用可能

### 改善
- **グローバル UI コンポーネント刷新**：ボタン、入力、ポップオーバー、サイドパネル、空状態、Toast の角丸・影・フォーカスリングを統一
- **AI サイドバー調整**：Q&A レイアウト、モデル/権限コントロール、四角い思考スピナー、入力タイポグラフィ
- **変更履歴ダイアログ再設計**：バージョン折りたたみ、セクション色分け、UI 言語に追従

## [0.5.0] - 2026-07-17

### 機能
- **ターミナル Hex/Raw ストリーム診断パネル**：オプションで有効化し、セッションの生の入出力をバイト単位で確認。エンコーディングやエスケープシーケンスの問題調査に便利
- **JSON ホストソース**：ローカル JSON ファイルまたは HTTP(S) エンドポイントからホストインベントリを取得（CMDB / Ansible / カスタム API 形式）。メタデータのみで、秘密情報を含むインベントリは拒否。HTTP 認証ヘッダーに対応
- **ホストインベントリの共有とインポート**：チーム引き継ぎ用にメタデータのみのインベントリをエクスポート（Ansible YAML 形式を含む）、クリップボードからインポート
- **名前付きワークスペーステンプレート**：ホストの割り当て、分割レイアウト、任意の cwd/起動コマンドをテンプレートとして保存し、クイックスイッチャーからワンクリックで適用
- **接続ログのブックマーク**：再生位置のブックマーク + メモ + 検索ジャンプ。ログ一覧にブックマーク数を表示
- **ポート転送のライブチャネルビュー**：ローカル/リモート/ダイナミック転送の接続ごとの送信元、宛先、トラフィックバイト統計
- **スクリプト onOutput トリガーアクション拡張**：出力パターンにマッチしたらデスクトップ通知、サウンド、タブマーカー、セッション録画開始を選択可能
- **セーフペーストと精密ブロードキャスト**：複数行ペーストの遅延 / プロンプト待ち / 危険コマンド確認。ブロードキャストはワークスペース/選択中/グループ/ウィンドウを正確に指定可能
- **システム OpenSSH チャネル強化**：GSSAPI/Kerberos とポスト量子（PQ）アルゴリズムをシステム OpenSSH 経由でサポート；ジャンプチェーンと HTTP/SOCKS プロキシに対応
- **内蔵 ssh2 ハイブリッド後量子 KEX**：`mlkem768x25519-sha256`（ML-KEM-768 + X25519）を優先し、未対応時は古典アルゴリズムへフォールバック；システム ssh 必須ではない
- **RDP ホスト対応**：Vault ホストで RDP を有効化し、システム遠隔デスクトップクライアント（Windows mstsc / macOS Windows App / Linux xfreerdp）を起動
- **更新履歴が UI 言語に追従**：アプリ内 Changelog を現在の UI 言語で表示（10 言語）

### Windows ARM64
- **win-arm64 インストーラーに mosh / ET を同梱**：MoshMagies 0.1.9 と EternalTerminal 6.2.10 が Windows arm64 ネイティブバイナリを初提供
- **win-arm64 専用の自動更新フィード**：更新メタデータは専用の `latest-arm64.yml` チャネルに移行し、x64 の更新に追従しなくなりました（従来は arm64 の更新で x64 パッケージがインストールされ、エミュレーションで動作していました）

## [0.4.5] - 2026-07-15

### 修正
- **入れ子暗号文による 401 / 空ストリーム**：キーチェーン障害中に繰り返し保存すると鍵が何重にも暗号化されていました（`enc:v2(enc:v1(...))`）。復号ループの境界を修正し、許容範囲内の多層ネストを完全に復号。「正しく復号したのに破棄」や復号失敗の誤報がなくなりました
- **単一の破損資格情報がストア全体の読み込みを壊さないように**：フィールドの復号失敗時は保存値をそのまま保持（fail-soft）し、ストアは通常どおり読み込み。キーチェーン修復後に鍵を復元可能
- **Web 検索 API キー**：復号失敗後、フォーカス/フォーカス喪失だけで保存済みキーが誤って消えなくなりました。復号/暗号化失敗の明確な通知を追加し、サイレントではなくなりました
- **Windows DPAPI 暗号文の識別を修正**：二重暗号化防止ガードが DPAPI 鍵（`AQAAAN` ヘッダー）を見逃し、キーチェーン障害時に入れ子暗号文へ再暗号化していた問題を修正
- **Cursor Agent**：復号失敗時に暗号文を API キーとして子プロセスへ注入しなくなりました
- 設定の Provider / Web 検索 / Cursor の 3 箇所を統一：復号失敗時はキーの再入力を明確に促し、UI 言語の切り替えで未保存のキーが上書きされなくなりました

## [0.2.0] - 2026-07-11

### 機能
- 自動更新の IPC イベントが単一ウィンドウにのみ送信される問題を修正し、全ウィンドウへブロードキャストに変更(メイン + 設定ウィンドウの両方が受信)
- 手動更新チェックと自動更新の状態機械を統一し、3 系統の並行状態を解消
- 手動「更新を確認」は GitHub API でバージョンを検出し、更新があれば非同期で electron-updater のダウンロードを起動
- 設定ウィンドウで「更新を確認」をクリック後、ダウンロード進捗が UI にリアルタイム反映
- アプリ起動 5 秒後に `electron-updater` の更新チェックを自動実行、手動クリック不要
- 新バージョン発見後に自動でダウンロード開始(`autoDownload=true`)
- ダウンロード完了後に永続トースト通知を表示、「今すぐ再起動」クリックでインストール
- ダウンロード失敗時はエラートーストを表示し、「Releases を開く」フォールバック入口を提供
- Settings > System の進捗バーが自動ダウンロード進捗をリアルタイム表示、`useUpdateCheck` が一元駆動
- electron-updater 非対応の Linux deb/rpm/snap などは自動スキップし、従来の GitHub API 通知動作を維持

### 設計メモ
- `broadcastToAllWindows` が `getSenderWindow` の単一送信を置き換え、全ウィンドウが IPC イベントを受信することを保証
- `manualCheckStatus` フィールドが手動チェックの UI 状態(idle/checking/available/up-to-date/error)を追跡し、UI 層で `autoDownloadStatus` と優先度に応じて描画
- `SettingsSystemTab` はローカルの update state を保持せず、`useUpdateCheck` の統一データを一方向で受信
- 従来の 2 つの独立系統(GitHub API 通知 + electron-updater 手動ダウンロード)を統一状態機械に統合:`useUpdateCheck` が唯一の情報源として `App.tsx` のトーストと `SettingsSystemTab` の進捗バーを駆動
- グローバル永続 IPC リスナーは `autoUpdateBridge.init()` で一度だけ登録し、手動ダウンロード要求のたびの登録/解除の繰り返しを回避
- `autoInstallOnAppQuit=false`:サイレントインストールは行わず、ユーザーが再起動を明示的に実行

### インターフェース変更（SettingsSystemTabProps）
- 削除:`autoDownloadStatus`、`downloadPercent`
- 追加:`updateState`(完全な UpdateState)、`checkNow`、`installUpdate`、`openReleasePage`

### 注意事項
- `checkNow` の意味:GitHub API(`performCheck`)で新バージョンを検出し、更新があり electron-updater が未ダウンロードの場合、非同期で `bridge.checkForUpdate()` を起動して自動ダウンロードフローを開始
- 本機能はパッケージ後のアプリ(Windows NSIS、macOS dmg/zip、Linux AppImage)でのみ有効。dev モードでは `forceDevUpdateConfig=true` + `dev-app-update.yml` でのテストが必要(`.gitignore` 参照)
- 旧 `hasUpdate` トーストは `autoDownloadStatus !== 'idle'` の間は自動抑制され、新トーストとの重複を回避

### CI / ビルド改善
- macOS / Windows ビルドをスキップ(有償コード署名証明書が必要)し、無償の Linux ディストリビューションパッケージ提供に集中
- Linux x64(AlmaLinux 8)コンパイラ更新:Clang を優先、gcc-toolset-13 へフォールバック
- Linux arm64(Debian Bullseye)コンパイラ更新:`build-essential` から `clang-14 + lld-14` へ
- Release ジョブは macOS/Windows ビルドに依存せず、tag プッシュ後に Linux 成果物から直接 Release を公開
- deb 成果物検証を緩和:ファイルが見つからない場合は error ではなく warning を出力し、プラットフォームスキップによる CI 失敗を回避
