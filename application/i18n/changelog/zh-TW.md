# 更新日誌


## [0.5.23] - 2026-07-22

### 修復
- **啟動主題腳本先前從未執行**：它負責在介面繪製前套用已儲存的主題、強調色與語言，但因內嵌寫法被 CSP 直接拒絕執行，導致啟動瞬間會閃出錯誤配色。現已移為獨立檔案，未放寬任何安全策略
- **frame-ancestors 改由回應標頭下發**：該指令寫在 `<meta>` 中會被瀏覽器忽略、實際不生效；現由 app:// 回應標頭與開發伺服器下發，並新增測試防止內嵌腳本再次出現

### 功能
- **Cast 錄製回放**：應用先前能錄製 asciinema cast v2 卻打不開，現支援播放/暫停/進度跳轉/1x·2x·4x 倍速；被中途截斷的錄製會跳過損壞行並提示條數，而非拒絕整個檔案
- **工作階段記錄內搜尋**：記錄檢視器支援 Cmd/Ctrl+F 檢索，與即時終端的搜尋互不干擾
- **Hex 面板每列位元組數**：可在 8 / 16 / 32 間切換，切換後已擷取內容立即重排
- **更新記錄依類別篩選**：安全 / 功能 / 修復 / 優化分類標籤，各顯示條目數

### 優化
- **清理無法生效的宣告**：移除一批定義後從未被呼叫的程式碼，包括一個從不驗證卻聲稱存在的團隊權限、一個設了永不生效的群組欄位，以及一份與主行程實作重複、永遠不會被載入的 WAN 邀請解析模組
- **書籤錨點測試加固**：補齊位元組偏移與行號互轉在 CRLF、越界、精確往返等邊界上的覆蓋

## [0.5.22] - 2026-07-21

### 安全
- **團隊稽核簽章真實驗證**：先前只要事件帶 `sig` 欄位就顯示勾號，從不驗證；現在逐筆 HMAC 驗簽，並區分「已驗證 / 遭竄改 / 未簽章 / 無金鑰可驗」四種狀態
- **探測不再回傳憑證**：Proxy 連線測試僅回傳錯誤碼而非原始訊息（HTTP CONNECT 會回顯 `Proxy-Authorization`，ProxyCommand 會回顯命令列）；資料來源欄位對應無法繞過清單的密鑰檢查

### 功能
- **SFTP 批次重新命名**：支援 `{name}` / `{ext}` / `{n}` 樣板與序號補零，執行前算出整批計畫並逐筆預覽；任一處重名或撞上未參與重新命名的檔案即整批拒絕
- **主機欄位批次編輯**：一次修改選取主機的使用者名稱、群組、連接埠與標籤；標籤為附加而非取代，留空表示維持原值；資料來源託管的主機會被排除並提示數量
- **結構化主機搜尋**：支援 `tag:` `user:` `group:` `host:` 篩選與自由文字混用；僅在冒號前為已知欄位名稱時才視為篩選器，IPv6 位址與含全形冒號的中文標籤不受影響
- **F1 快速鍵速查表**：可搜尋、依分類分組，讀取目前實際生效的按鍵綁定；本身也是一條可改鍵的一般快速鍵
- **序列埠十六進位傳送**：按原樣寫入裝置，繞過字元集編碼層，附位元組預覽與 ASCII 對照
- **資料來源欄位對應**：可將 `name` / `ip` / `ssh_port` 之類的欄位名稱對應到規範欄位，供無法修改的外部清單使用
- **接通既有能力**：tmux 向視窗傳送指令、Proxy 連線測試、匯入金鑰時一併辨識憑證、PKCS#11 模組路徑預設

### 修復
- **團隊 Vault 匯入不再假成功**：先前匯入分享包會提示「已匯入 N 台主機」，但主機被丟棄、清單毫無變化；現在真正寫入 Vault 並回報實際落地數量，且只增不覆蓋本機變更與憑證
- **WAN 邀請碼可以加入了**：`magies-follow:2:` 邀請一律被當作 LAN 解析並回報 `version` 錯誤，儘管介面明寫支援；現在依邀請版本分流傳輸層
- **主機匯出遵守勾選**：勾選部分主機後匯出仍會匯出全部
- **下拉選單長標籤不再溢出**：固定寬度導致最長的項目文字跑出選單、圖示被擠成零寬；排序選單在法文下有三個選項溢出
- **tmux 傳送指令可用**：該操作要求一個介面無法提供的窗格序號，導致功能存在卻無法觸發

### 優化
- **SFTP 傳輸顯示剩餘時間**：速率未知時不佔位；目錄列不顯示（其總量以檔案數計）
- **頂列顯示同步新鮮度**：停留即可看到資料上次真正落地的時間，無需展開面板
- **AI 對話匯出保留思考過程**：Markdown 摺疊呈現，純文字縮排呈現，並附耗時
- **AI 設定體驗**：快捷訊息支援搜尋；本機模型未偵測到工具呼叫時說明其影響
- **當機回報可見傳送量**：僅在上傳成功後計數，失敗與去重不計入
- **行程與 tmux 操作改用應用內確認框**：不再使用系統原生彈窗，跟隨應用主題
- **行動端納入品質關卡**：`mobile/` 先前被 lint 與測試完全排除，其測試從未被 CI 執行過

## [0.5.21] - 2026-07-21

### 安全

- **Kubernetes 維運擴展**：結構化 Events、Rollout 狀態/歷史/重啟、非互動 Agent Exec、互動終端 Exec 與回環 Port Forward

### 修正

- **跨平台發佈包**：修正免費無 Developer ID 的 Apple Silicon Fuse 後完整性閃退；Android 統一 JDK 21、SDK/Build Tools 36 與 v0.5.21 中繼資料

### 設計原理
- `broadcastToAllWindows` 替換 `getSenderWindow` 單點傳送，保證所有視窗都能收到 IPC 事件
- `manualCheckStatus` 欄位追蹤手動檢查 UI 狀態（idle/checking/available/up-to-date/error），與 `autoDownloadStatus` 在 UI 層按優先級渲染
- `SettingsSystemTab` 不再持有本機 update state，單向接收 `useUpdateCheck` 統一資料
- 將原有兩套獨立系統（GitHub API 通知 + electron-updater 手動下載）合併為統一狀態機：`useUpdateCheck` 作為唯一事實來源，同時驅動 `App.tsx` toast 和 `SettingsSystemTab` 進度條
- 全域持久化 IPC 監聽器在 `autoUpdateBridge.init()` 時一次性註冊，避免每次手動下載請求重複註冊/清理監聽器
- `autoInstallOnAppQuit=false`，不做靜默安裝，由使用者主動觸發重啟

### 注意事項
- `checkNow` 語義：使用 GitHub API（`performCheck`）偵測是否有新版本，若發現更新且 electron-updater 尚未開始下載，則非同步觸發 `bridge.checkForUpdate()` 啟動自動下載流程
- 此功能僅對打包後的應用（Windows NSIS、macOS dmg/zip、Linux AppImage）生效，dev 模式需配合 `forceDevUpdateConfig=true` + `dev-app-update.yml` 測試（見 `.gitignore`）
