## [0.5.0] - 2026-07-17

### Функции
- **Панель диагностики Hex/Raw-потока терминала**: включается по желанию, показывает необработанный ввод/вывод сессии побайтово — удобно для разбора проблем с кодировками и escape-последовательностями
- **JSON-источник хостов**: загрузка инвентаря хостов из локального JSON-файла или HTTP(S)-эндпоинта (стиль CMDB / Ansible / собственного API); только метаданные, инвентари с секретами отклоняются; поддерживаются заголовки HTTP-аутентификации
- **Обмен инвентарём хостов и импорт**: экспорт инвентаря только с метаданными для передачи между командами (включая формат Ansible YAML), импорт из буфера обмена
- **Именованные шаблоны рабочих пространств**: сохраняйте привязки хостов, раскладки разделения и необязательные cwd/команды запуска как шаблоны и применяйте их в один клик из быстрого переключателя
- **Закладки журналов подключений**: закладки позиции воспроизведения + заметки + переход по поиску; список журналов показывает число закладок
- **Живой просмотр каналов проброса портов**: источник, назначение и статистика байтов трафика по каждому соединению для локальных/удалённых/динамических пробросов
- **Расширение действий триггера onOutput в скриптах**: при совпадении шаблона вывода можно вызвать уведомление на рабочем столе, звук, отметку вкладки или запуск записи сессии
- **Безопасная вставка и точная трансляция**: задержка многострочной вставки / ожидание приглашения / подтверждение опасных команд; трансляцию можно точно нацелить на рабочее пространство/выбранное/группу/окно
- **Улучшения каналов системного OpenSSH**: GSSAPI/Kerberos и постквантовые (PQ) алгоритмы поддерживаются через системный OpenSSH; доступны jump-цепочки и HTTP/SOCKS-прокси
- **Встроенный гибридный постквантовый KEX ssh2**: предпочитает `mlkem768x25519-sha256` (ML-KEM-768 + X25519) с откатом к классическим алгоритмам; системный ssh больше не обязателен
- **Поддержка RDP-хостов**: включить RDP для хостов Vault и запускать системный клиент удалённого рабочего стола (Windows mstsc, macOS Windows App, Linux xfreerdp)
- **Журнал изменений следует языку интерфейса**: заметки о выпуске на текущем языке UI (10 языков)

### Windows ARM64
- **Установщики win-arm64 теперь включают mosh / ET**: MoshMagies 0.1.9 и EternalTerminal 6.2.10 впервые выходят как нативные бинарники Windows arm64

## [0.4.5] - 2026-07-15

### Исправления
- **401 / пустые потоки из-за вложенного шифротекста**: повторные сохранения во время сбоя связки ключей заворачивали ключи в слои шифрования (`enc:v2(enc:v1(...))`); после исправления границы цикла дешифрования многослойные вложения в пределах бюджета расшифровываются полностью — больше нет «расшифровал и выбросил» и ложных ошибок дешифрования
- **Одна испорченная запись больше не рушит загрузку всего хранилища**: при сбое дешифрования поля сохранённое значение остаётся как есть (fail-soft), хранилище загружается штатно, а ключи можно восстановить после ремонта связки ключей
- **API-ключ веб-поиска**: после сбоя дешифрования простое фокусирование/расфокусирование больше не стирает сохранённый ключ; добавлены явные уведомления о сбоях шифрования/дешифрования вместо молчания
- **Исправлено распознавание шифротекста Windows DPAPI**: защита от повторного шифрования раньше пропускала ключи DPAPI (заголовок `AQAAAN`), и сбой связки ключей превращал их во вложенный шифротекст; исправлено
- **Cursor Agent**: при сбое дешифрования шифротекст больше не передаётся дочернему процессу как API-ключ
- Унификация трёх зон настроек Provider / веб-поиск / Cursor: сбой дешифрования теперь явно просит ввести ключ заново, а смена языка интерфейса не затирает несохранённый ключ

## [0.4.4] - 2026-07-14

### Исправления
- **ИИ 401 / пустые потоки**: если дешифрование API-ключа не удалось или ключ не синхронизирован с главным процессом, запросы больше не уходят к провайдерам с заглушкой `__IPC_SECURED__`; они сразу завершаются ошибкой с просьбой сохранить ключ заново
- Перед отправкой сообщений ожидается синхронизация провайдеров с главным процессом, что исключает сбои аутентификации из-за гонок
- Понятные подсказки по аутентификации, когда локальный ключ непригоден (сбой дешифрования / отсутствует / осталась заглушка)

## [0.2.0] - 2026-07-11

### Функции
- Исправлена отправка IPC-событий автообновления только одному окну; теперь они транслируются всем окнам (главное + настройки получают оба)
- Унифицированы конечные автоматы ручной проверки и автообновления, устранены три параллельных состояния
- Ручная «Проверка обновлений» определяет версии через GitHub API и при находке асинхронно запускает загрузку electron-updater
- После клика «Проверить обновления» в окне настроек прогресс загрузки отражается в интерфейсе в реальном времени
- Приложение автоматически запускает проверку `electron-updater` через 5 секунд после старта, без ручного клика
- При обнаружении новой версии загрузка начинается автоматически (`autoDownload=true`)
- По завершении загрузки появляется постоянный тост; клик «Перезапустить сейчас» устанавливает обновление
- При сбое загрузки появляется тост ошибки с запасным пунктом «Открыть Releases»
- Полоса прогресса в Settings > System показывает автозагрузку в реальном времени, единообразно управляясь `useUpdateCheck`
- Платформы Linux deb/rpm/snap и другие без поддержки electron-updater пропускаются автоматически, сохраняя прежнее поведение уведомлений через GitHub API

### Заметки о дизайне
- `broadcastToAllWindows` заменяет одиночную отправку `getSenderWindow`, гарантируя получение IPC-событий всеми окнами
- Поле `manualCheckStatus` отслеживает состояние интерфейса ручной проверки (idle/checking/available/up-to-date/error) и отрисовывается вместе с `autoDownloadStatus` по приоритету
- `SettingsSystemTab` больше не держит локальное состояние обновления; он однонаправленно получает единые данные `useUpdateCheck`
- Две прежде независимые системы (уведомления GitHub API + ручная загрузка electron-updater) объединены в один конечный автомат: `useUpdateCheck` — единственный источник истины, управляющий и тостом `App.tsx`, и полосой прогресса `SettingsSystemTab`
- Глобальные постоянные IPC-слушатели регистрируются один раз в `autoUpdateBridge.init()`, избегая повторной регистрации/очистки при каждом ручном запросе загрузки
- `autoInstallOnAppQuit=false`: без тихой установки, перезапуск инициирует пользователь

### Изменения интерфейса（SettingsSystemTabProps）
- Удалены: `autoDownloadStatus`, `downloadPercent`
- Добавлены: `updateState` (полный UpdateState), `checkNow`, `installUpdate`, `openReleasePage`

### Примечания
- Семантика `checkNow`: определяет новые версии через GitHub API (`performCheck`); если обновление есть и electron-updater ещё не начал загрузку, асинхронно вызывается `bridge.checkForUpdate()` для запуска автозагрузки
- Функция работает только в упакованных приложениях (Windows NSIS, macOS dmg/zip, Linux AppImage); режим dev требует `forceDevUpdateConfig=true` + `dev-app-update.yml` для тестов (см. `.gitignore`)
- Старый тост `hasUpdate` подавляется, пока `autoDownloadStatus !== 'idle'`, исключая дублирование с новым тостом

### Улучшения CI / сборки
- Пропущены сборки macOS / Windows (нужны платные сертификаты подписи кода) в пользу бесплатных пакетов Linux
- Обновление компилятора Linux x64 (AlmaLinux 8): предпочтителен Clang, откат на gcc-toolset-13
- Обновление компилятора Linux arm64 (Debian Bullseye): с `build-essential` на `clang-14 + lld-14`
- Задача релиза больше не зависит от сборок macOS/Windows; пуш тега публикует релиз напрямую из артефактов Linux
- Смягчена проверка артефактов deb: отсутствующие файлы дают предупреждение вместо ошибки, чтобы пропуски платформ не валили CI
